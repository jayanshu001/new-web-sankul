# Deep-Linking — Share URLs

How the backend generates `shareableLink` values, and what the mobile app must register to open them.

## Backend: how a share URL is built

Generated by `buildShareUrl(resource, id, fallbackBase)` in [src/deeplinking/shareRedirect.ts](../src/deeplinking/shareRedirect.ts).

```
{SHARE_BASE_URL or request origin}/share/{resource}/{id}
```

Example: `https://api.gpscvideo.com/share/courses/665aabbc1122334455667788`

The route is registered in [src/deeplinking/deeplinking.routes.ts](../src/deeplinking/deeplinking.routes.ts). When a user opens the link, the backend serves an HTML page (see [src/deeplinking/templates/share-redirect.html](../src/deeplinking/templates/share-redirect.html)) that attempts:

1. Open the app via custom scheme: `{APP_SCHEME}://{deep-path}/{id}`
2. Fall back to the universal/web link: `{APP_WEB_HOST}/{deep-path}/{id}`
3. Fall back to the Play Store / App Store
4. Final fallback: `SHARE_FALLBACK_URL`

## Resource slugs and in-app deep paths

| Resource    | Public URL slug  | In-app deep path | Source                                                                 |
|-------------|------------------|------------------|------------------------------------------------------------------------|
| Course      | `courses`        | `course`         | `GET /share/courses/:id`                                               |
| Book        | `books`          | `book`           | `GET /share/books/:id`                                                 |
| Ebook       | `ebooks`         | `ebook`          | `GET /share/ebooks/:id`                                                |
| Live Course | `live-courses`   | `live-course`    | `GET /share/live-courses/:id`                                          |
| Package     | `packages`       | `package`        | `GET /share/packages/:id`                                              |
| Test Series | `test-series`    | `test-series`    | `GET /share/test-series/:id`                                           |
| Educator    | `educators`      | `educator`       | `GET /share/educators/:id`                                             |

The **public URL slug** is what appears in the shared link. The **in-app deep path** is what the mobile app must recognize.

## What mobile needs to handle

For each resource above, register both:

1. **Custom scheme** — e.g. `com.gpscvideo.gpsc://course/<id>`
2. **Universal/App link** — e.g. `https://com.gpscvideo.com/course/<id>`

When the OS hands the app a path matching `<deep-path>/<id>`, route the user to the corresponding screen with that `id`.

| Deep path        | Expected screen                  |
|------------------|----------------------------------|
| `course/<id>`    | Recorded course detail           |
| `book/<id>`      | Physical book detail             |
| `ebook/<id>`     | Ebook detail                     |
| `live-course/<id>` | Live course detail             |
| `package/<id>`   | Package detail                   |
| `test-series/<id>` | Test series detail             |
| `educator/<id>`  | Educator profile + their courses |

The `<id>` is always a 24-char MongoDB ObjectId — the backend already validates this on `/share/...`, so anything that reaches the app is well-formed.

## API: where `shareableLink` appears in responses

Backend embeds the share URL directly in response payloads — the client/share button can just read it; it does not need to construct anything.

### Courses
- `GET /api/v1/client/courses` — `data[].shareableLink`
- `GET /api/v1/client/courses/categories/:categoryId/courses` — `data[].shareableLink`
- `GET /api/v1/client/courses/:id` — `data.shareableLink` (top-level)

### Books
- `GET /api/v1/client/books` — `data.books[].shareableLink`
- `GET /api/v1/client/books/trending` — `data.items[].shareableLink` (slug is `books` or `ebooks` per item `type`)
- `GET /api/v1/client/books/:id` — `data.shareableLink`

### Ebooks
- `GET /api/v1/client/ebooks` — `data.ebooks[].shareableLink`
- `GET /api/v1/client/ebooks/subscriptions` — `data.subscriptions[].shareableLink`
- `GET /api/v1/client/ebooks/:id` — `data.ebook.shareableLink`

### Live courses
- `GET /api/v1/client/live-courses` — `data.liveCourses[].shareableLink`
- `GET /api/v1/client/live-courses/:id` — `data.shareableLink` and `data.liveCourse.shareableLink`
- `GET /api/v1/client/live-courses/my` — `data.liveCourses[].liveCourse.shareableLink`

### Packages
- `GET /api/v1/client/packages` — `data[].shareableLink`
- `GET /api/v1/client/packages/type/:typeId` (and similar list variants) — `data[].shareableLink`
- `GET /api/v1/client/packages/goal` — `data[].label.packages[].shareableLink`
- `GET /api/v1/client/packages/:id` — `data.package.shareableLink`
- `GET /api/v1/client/packages/my` — `data[].packageId.shareableLink`

### Test series
- `GET /api/v1/client/test-series` — `data.data[].shareableLink`
- `GET /api/v1/client/test-series/:id` — `data.shareableLink` and `data.series.shareableLink`
- `GET /api/v1/client/test-series/my/subscriptions` — `data.data[].testSeriesId.shareableLink`

### Educators
- `GET /api/v1/client/educators/:id` — `data.shareableLink`, `data.educator.shareableLink`, and `data.courses[].shareableLink` (each course link uses the `courses` slug)

## Environment variables

Configured per-environment. The backend reads them once at startup.

| Variable              | Purpose                                                          | Example                                                |
|-----------------------|------------------------------------------------------------------|--------------------------------------------------------|
| `SHARE_BASE_URL`      | Origin used inside generated share URLs                          | `https://api.gpscvideo.com`                            |
| `ORIGIN`              | Fallback for `SHARE_BASE_URL` when not set                       | `https://api.gpscvideo.com`                            |
| `APP_SCHEME`          | Custom URL scheme registered by the mobile app                   | `com.gpscvideo.gpsc`                                   |
| `APP_WEB_HOST`        | HTTPS host registered for universal/app links                    | `https://com.gpscvideo.com`                            |
| `PLAY_STORE_URL`      | Android store fallback                                           | `https://play.google.com/store/apps/details?id=...`    |
| `APP_STORE_URL`       | iOS store fallback                                               | `https://apps.apple.com/us/app/gpsc/id6751284655`      |
| `SHARE_FALLBACK_URL`  | Final fallback when nothing else opens                           | `https://www.gpscvideo.com/`                           |

## Universal-link verification files

Already served (see commit `028d64f`):

- iOS: `/.well-known/apple-app-site-association`
- Android: `/.well-known/assetlinks.json`

When adding new deep paths, no change to these files is required — they apply to all paths under `APP_WEB_HOST` by default. Confirm with mobile if their AASA/assetlinks intent filter restricts paths.

## Adding a new shareable resource

1. Add a route in [src/deeplinking/deeplinking.routes.ts](../src/deeplinking/deeplinking.routes.ts):
   ```ts
   router.get("/<slug>/:id", sendShare("<deep-path>"));
   ```
2. In the relevant controller, attach `shareableLink: buildShareUrl("<slug>", id, baseUrl)` to the response.
3. Tell mobile the new `<deep-path>` so they can register an intent filter / universal link handler for it.
